# Documentation for Visas

The idea here is that most different countries are asking for essentially
the same information, but they all use different forms and the answers go
in different places.

So imagine one (gullible) person going through lots of different countries
but instead of filling the forms out directly, instead, noting each place
it asks for information and what it asks for in a database.

Then if they can ANSWER those questions and it will create fully filled out forms
but the NEXT person who comes through only has to answer the questions.

If the questions are the same from form to form, people are only answering
them once and potentially neatly filling out dozens of Visas (other than
signing them).

It saves no time if only one person uses it

But if people started to use it, the time-saving would be exponential.
With a large enough constituency, you can imagine working with different countries
to make their visa apps "compatible", which is to say, getting the newest
Visa applications up front, along with the locations and questions.

But, we don't live in that world.
